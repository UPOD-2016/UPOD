1. Introduction

1.1 Scope

UPOD's web design promotes positive user experience, given that current physics resources (i.e. HyperPhysics) feature designs that are not user friendly. As such, the front end of UPOD will emphasize ease of navigation.

1.2 Definitions, Acronyms, and Abbreviations

UPOD

  • Undergraduate Physics Online Database

    • Web Design

  • Pages, features, and functions that users will interact with and make use of while navigating the UPOD site.

    • Moderator

  • An individual with permission to add/delete/edit categories and articles of the UPOD site.

    • User

  • An individual who can only view content of the site.

    • Category (/Sub-Category)

  • A set of Article pages that fall into a definable block of information (i.e. Classical Mechanics or Optics)

    • Article

  • Viewable page containing information, formulas, and diagrams related to a single concept.

    • Diagram

  • Illustration of physics concept. Can be static or dynamic, as well as support interaction.

    • MySQL

  • Open source relational database management system.

    • SVG

  • Scalable Vector Diagram - image format for diagrams with support for animation and interaction.

    • UML

  • Unified Markup Language - modeling language to visualize design of a system.

    • AD

  • Active Directory - moderator login verification through LDAP in order to allow moderators to access restricted operations.

    • LDAP

  • Lightweight Directory Access Protocol - protocol to manage individuals and files in a network.

    • Development Environment

  • Test bed for modifications for the site. Only viewable by moderators.

    • Production Environment

  • Live content of the website.

    • 2. Overall Description

    2.1 Product Perspective

    Web Design relates to the front end of the UPOD system - the components of the website that users will interact with. Since the front end is only a part of the entire system, other features supported by the site are outlined in different requirement documents. Interface between the various sections of the project is under section 2.1.3 of this requirement document. More detail will be added during the analysis phase.

    2.1.1 System Interfaces

    UPOD content operates on modern and standard browsers.

  • Google Chrome - version 51
  • Mozilla Firefox - version 47
  • Internet Explorer - version 11
  • Microsoft Edge - version 38
  • Safari - version 9.1

    • 2.1.2 User Interfaces

    Users are expected to interact with UPOD with a mouse and keyboard, and priority will be given to ensuring proper interaction via this method. Interaction with UPOD by touchscreen will be added if time allows.

    2.1.3 Software Interfaces

  • The front end will contain information contained in a MySQL database. Animations and diagrams displaying physics content is to be implemented using animation software (SVG).

    • 2.1.4 Communications Interfaces

  • UPOD will remain responsive across internet service providers. More detail to be added later with respect to Laurier network restrictions and accessible ports.

    • 2.1.5 Memory

  • No memory constraints identified at this point for the front end. Expected memory constraints for back end and diagrams/animation content is located in their respective requirement documents.

    • 3. Specific Requirements

    3.1 External Interfaces

  • None identified at this point.

    • 3.2 Functions

  • UPOD must be searchable by text, as well as support navigation through site content, such as by the category/article structure.
  • UPOD's web interface will contain both static and animated diagrams that users can view to improve their understanding of concepts.
  • UPOD will allow for site moderators to add, delete, and modify pages and categories when required. These modifications to the site content will be done through a graphical user interface.

    • 3.3 Performance Requirements

  • Searches should be satisfied within two seconds of the search query. More detail will be added at a later date. Performance of the site under heavy load will not be a concern.

    • 3.4 Logical Database Requirements

  • All physics content and diagrams will be stored in a relational database. More information pertaining to database structure can be found in the Back End requirement document.

    • 3.5 Software System Attributes

    3.5.1 Reliability

  • UPOD will be expected to be available as a permanent reference tool, with maintenance conducted in a development environment and copied to production environment live.

    • 3.5.2 Availability

  • Software used in UPOD's front end must be supported across the previously listed browsers, and commonly available for users to set up if required. (i.e. if their browser requires a plug-in or runtime to display diagrams, etc.)

    • 3.5.3 Security

  • All physics content on UPOD is free for all to use, and therefore there are no major security constraints. Moderator login and authorization will be handled by the WLU AD.

    • 3.5.4 Maintainability

  • Code should follow standard protocols and be well documented for future maintainability. UPOD is assumed to be available for the foreseeable future, therefore the set of individuals in charge of maintenance will not be constant.

    • 3.5.5 Portability

  • Development will take place on a Unix based server, so no portability concerns are identified. Any reasonable requirements for software to be installed on Hopper will be communicated to Mr. David Brown as soon as they are identified and validated.

    • 3.6.4 Pages and Features

    Landing Page

  • The UPOD Landing Page is the first thing users will see when accessing UPOD. The UPOD design philosophy will encourage users to search the site quickly - the focus on and size of the search bar on the landing page will promote a quick search for whatever term the user is looking for. (i.e. - similar to the Google Search page.)

    • Category Page

  • The Category page will act as a template for the splash page of each physics discipline. Each discipline is to be shown on its own Category page, which will display a summary of the physics discipline as well as contain a list of all Article pages that fall under that discipline.

    • Article Page

  • The Article page will act as a template for creation of the majority of UPOD's content. Each concept or law shown on its own Article page will display background information, important formulas, and link users to relevant information. Relevant links will be displayed at the bottom of the page, similar to the "See Also" section of Wikipedia pages.

    • About UPOD Page

  • This page will outline the purpose of UPOD: making accurate and easy to understand physics content accessible to undergraduate students. It will also contain a section to give credit to those who contributed to the creation of the site.
  • This is also where users will be able to contact UPOD moderators by email to report missing, incorrect, or broken content.

    • Moderator Login Page

  • Site moderators will need to enter credentials on this page to gain access to the site's edit mode - the URL for this page will be given only to moderators (i.e. no direct link to moderator portal.) Successful login will allow the moderator to access the UPOD development environment

    • Search Results Page

  • Topic searches should contain enough relevant information to satisfy the user's search. Since some information can apply to many different disciplines, (i.e. Speed of Light in both Classical Mechanics as well as Optics) a search for terms should list all Article pages that refer to that term. Results should be displayed with an Article page heading as well as a brief preview of the info contained on the Article page. (i.e. similar to a general Google Search)

    • Search Bar

  • UPOD will features a search bar that is always accessible. It will be featured in the center of the landing page, and present in the top right hand corner all other web pages. Searches will return results on the Search Results Page.

    • Slide Menu

  • The menu will also be accessible from each area of the website. It will feature links to each Category page, as well as a link to the Landing Page and the About UPOD page.

    • Diagrams

  • Various Article pages will feature diagrams that illustrate concepts. Diagrams will be created in SVG, and be either static or dynamic. Dynamic Diagrams may also support user interaction. Further information can be found in the Graphics and Animation requirements document.
