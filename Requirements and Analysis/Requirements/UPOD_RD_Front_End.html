1. Introduction

1.1 Scope

UPOD's web design promotes positive user experience, given that current physics resources (i.e. HyperPhysics) feature poor designs. As such, the front end of UPOD will emphasize user friendly navigation.

1.2 Definitions, Acronyms, and Abbreviations

UPOD

  • Undergraduate Physics Online Database

    • Web Design

  • Pages, features, and functions that users will interact with and make use of while navigating the UPOD site.

    • Moderator

  • An individual with permission to add/delete/edit categories, subcategories and articles of the UPOD site.

    • User

  • An individual who can only view content of the site.

    • Category (/Sub-Category)

  • A set of Article pages that fall into a definable block of information (i.e. Classical Mechanics or Optics)

    • Article

  • Viewable page containing information, formulas, and diagrams related to a single concept.

    • Diagram

  • Illustration of physics concept. Can be static or dynamic, as well as support interaction.

    • MySQL

  • Open source relational database management system.

    • SVG

  • Scalable Vector Diagram - image format for diagrams with support for animation and interaction.

    • UML

  • Unified Markup Language - modeling language to visualize design of a system.

    • AD

  • Active Directory - moderator login verification through LDAP in order to allow moderators to access restricted operations.

    • LDAP

  • Lightweight Directory Access Protocol - protocol to manage individuals and files in a network.

    • Development Environment

  • Test bed for modifications for the site. Only viewable by moderators.

    • Production Environment

  • Live content of the website.

    • GUI

  • Graphical User Interface.

    • 2. Overall Description

    2.1 Product Perspective

    Web Design relates to the front end of the UPOD system - the components that users will interact with on the website. Since the front end is only a part of the entire system, other features supported by the site are outlined in different requirement documents. Interface between the various sections of the project is under section 2.1.3 of this requirement document. More detail will be added during the analysis phase and as the project develops.

    2.1.1 System Interfaces

    UPOD content operates on modern and standard browsers.

  • Google Chrome - version 51
  • Mozilla Firefox - version 47
  • Internet Explorer - version 11
  • Microsoft Edge - version 38
  • Safari - version 9.1

    • 2.1.2 User Interfaces

  • Users are expected to interact with UPOD with a mouse and keyboard, and priority will be given to ensure proper interaction via this method.
  • Interaction with UPOD by touchscreen for Android and iPhone will be added if time allows.

    • 2.1.3 Software Interfaces

  • The front end will contain information contained in a MySQL database. Animations and diagrams displaying physics content is to be implemented using animation software (SVG).

    • 2.1.4 Communications Interfaces

  • UPOD will remain responsive across internet service providers.
  • More detail in Analysis phase with respect to Laurier network restrictions and accessible ports.

    • 2.1.5 Memory

  • Memory constraints will be at the very top 50MB for each page.
  • Estimated this amount by observing the average amount of memory used by other websites (i.e. Wikipedia, Facebook).

    • 2.2 Actors and Use Cases

  • As of now there are only two actors.

    • 2.2.1 Actors

    Moderator

  • Login/Logout
  • Add/Delete/Modify Categories, Sub-Categories and Articles

    • User

  • Search for Articles or Categories
  • View Articles or Categories

    2.3 Constraints

  • UPOD Web Design has a hard deadline by the end of the 2016 Spring Term (August 10, 2016).
  • The UPOD budget is $0, so any software/frameworks must be free to use.

    • 3. Specific Requirements

    3.1 Functions

  • UPOD must be searchable by text and support navigation through site content (i.e. category(s)/subcategory(s)/articles structure).
  • UPOD's web interface will contain both static and animated diagrams that users can view to improve their understanding of concepts.
  • UPOD will allow for site moderators to add, delete, and modify articles, categories and subcategories when required. These modifications to the site content will be done through a GUI.
  • A built in login process for site moderators to be utilized on UPOD, preventing unauthorized modifications and edits.

    • 3.2 Performance Requirements

  • Searches should be satisfied within two seconds of the search query.

    • 3.3 Software System Attributes

    3.3.1 Reliability

  • UPOD expected to be available as a permanent reference tool, with maintenance conducted in a development environment and copied to production environment live.

    • 3.3.2 Availability

  • Software used in UPOD's front end must be supported across the previously listed browsers, and commonly available for users to set up if required. (i.e. if their browser requires a plug-in or runtime to display diagrams, etc.)

    • 3.3.3 Security

  • All physics content on UPOD is free for all to use, and therefore there are no major security constraints. Login page for moderators will be available through a URL address known to moderators only. Login information and authorization will be handled by the WLU AD.

    • 3.3.4 Maintainability

  • Code should follow Google HTML/CSS style standards and be well documented for future maintainability. UPOD is assumed to be available for the foreseeable future, therefore the set of individuals in charge of maintenance will not be constant.

    • 3.3.5 Portability

  • Development will take place on a Unix based server. UPOD will always be on Hopper and therefore any reasonable requirements for software to be installed will be communicated to Mr. David Brown as soon as they are identified and validated.

    • 3.4 Organizing Specific Requirements

    3.4.1 System Mode

  • UPOD's functionality will be the same for all users, however, there will be an additional moderator mode (development environment) where content can be added, removed, or edited.

    • 3.4.2 Features

    Landing Page

  • The UPOD Landing Page is the first thing users will see when accessing UPOD. The UPOD design philosophy encourages users to search the site quickly - the focus on and size of the search bar on the landing page promotes a quick search for whatever term the user is looking for. (i.e. - similar to the Google Search page.)

    • Category Page

  • The Category page acts as a template for the splash page of each physics discipline. Each discipline is shown on its own Category page, which displays a summary of the physics discipline as well as contains a list of all Article pages that fall under that discipline.

    • Article Page

  • The Article page acts as a template for creation of the majority of UPOD's content. Each concept or law shown on its own Article page displays background information, important formulas, and links users to relevant information. Relevant links are displayed at the bottom of the page, similar to the "See Also" section of Wikipedia pages.

    • About UPOD Page

  • This page outlines the purpose of UPOD: making accurate and easy to understand physics content accessible to undergraduate students. It also contains a section to give credit to those who contributed to the creation of the site.
  • This is also where users are able to contact UPOD moderators by email to report missing, incorrect, or broken content.

    • Moderator Login Page

  • Site moderators need to enter credentials on this page to gain access to the site's edit mode - the URL for this page is given only to moderators (i.e. no direct link to moderator portal.) Successful login allows the moderator to access the UPOD development environment

    • Search Results Page

  • Topic searches contain enough relevant information to satisfy the user's search. Since some information can apply to many different disciplines, (i.e. Speed of Light in both Classical Mechanics as well as Optics) a search for terms should list all Article pages that refer to that term. Results should be displayed with an Article page heading as well as a brief preview of the info contained on the Article page. (i.e. similar to a general Google Search)

    • Search Bar

  • UPOD features a search bar that is always accessible. It is featured in the center of the landing page, and present in the top right hand corner all other web pages. Searches return results on the Search Results Page.

    • Slide Menu

  • The menu is accessible from each area of the website. It features links to each Category page, as well as a link to the Landing Page and the About UPOD page.

    • Diagrams

  • Various Article pages feature diagrams that illustrate concepts. Diagrams are created in SVG, and either static or dynamic. Dynamic Diagrams also support user interaction. Further information can be found in the Graphics and Animation requirements document.
