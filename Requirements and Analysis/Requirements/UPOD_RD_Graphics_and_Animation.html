1. Introduction

1.1 Scope

The UPOD Graphics and Animation group creates animations and interactive diagrams with SVG which allows users to gain a better understanding of Physics concepts. The specific diagrams that are animated are decided in tandem with the Physics Research group.

1.2 Definitions, Acronyms, and Abbreviations

G&A

  • Graphics and Animations

    • MVC

  • a software architectural pattern for implementing user interfaces on computers. Traditionally used for GUIs.

    • Model (MVC)

  • Represents the equation or physics concept illustrated.

    • View (MVC)

  • Sliders, buttons, and other UI elements to interact with the model. Objects to reflect inputs and demonstrate the concepts.

    • Controller (MVC)

  • Liases and controls the view to reflect the model.

    • SVC

  • Scalable Vector Graphics.

    • 1.3 References

  • IEEE Std 610.12-1990, IEEE Standard for Software Project Management Plans: http://bohr.wlu.ca/cp317/notes/IEEE_830.pdf
  • WLU CP317 Class of Spring 2013: Pong Tracker used as Example

    • 1.4 Overview

    The UPOD Graphics/Animation Requirement document outlines the following:

  • Overall description of the G&A
  • User's interaction with the G&A
  • The software used to create the G&A

    • 2. Overall Description

    2.1 Product Perspective

    G&A uses the information gathered by the Physics Research group to create diagrams and animations. In addition, both groups collaborate to decide on what physics animations should be made and also what type of physics diagrams should be used.

    2.1.1 System Interfaces

    The graphics and animations will operate on the following browsers

  • Google Chrome
  • Mozilla Firefox
  • Internet Explorer
  • Safari

    • 2.1.2 User Interfaces

  • Users can use a mouse to interact with certain animations or diagrams on UPOD.
  • The keyboard can be used to input values to certain diagrams.

    • 2.1.3 Software Interfaces

  • Animations and graphics displaying the physics content are created using javascript SVG.

    • 2.1.4 Communications Interfaces

  • Graphics and animations in UPOD is portable across internet browsers.

    • 2.1.5 Memory

  • Memory constraints should be expected while creating the graphics and animations. More information shall be added later.

    • 2.2 Constraints

  • UPOD graphics and animations must be added onto the site before the end of the 2016 spring term (July 26, 2016).
  • Budget is $0 so any software used to create the graphics and animations must be free to use.

    • 3. Specific Requirements

    3.1 External Interfaces

  • Different javascript SVG animation libraries are being researched.

    • 3.2 Functions

  • Graphics and animations to be included on the UPOD web interface.

    • 3.3 Performance Requirements

  • Graphics and animations should be able to be seen on different browsers.
  • More detail in Analysis phase.

    • 3.4 Logical Database Requirements

  • All physics content and graphics will be stored on a relational database (see Back End Requirements).

    • 3.5 Software System Attributes

    3.5.1 Reliability

  • UPOD Graphics and Animations are expected to be used permanently in the site.

    • 3.5.2 Availability

  • Software used to create Graphics and Animations must be supported across the previously listed browsers.

    • 3.5.3 Security

  • All Graphics and animations on UPOD are free to use for all and therefore are no major security constraints.

    • 3.5.4 Maintainability

  • Graphics and animations should be coded to standard protocols and should be well documented in order for future maintenance. UPOD is to be assumed available for the future; therefore having a set of individuals in charge of maintenance will not be constant.

    • 3.5.5 Portability

  • Graphics and animation will be developed on UNIX based software. Any new software to be installed on hopper will be communicated to an Administrator once they are identified and validated.
