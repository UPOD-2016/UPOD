    1. Documentation and implementation will be reviewed/tested by SQA team.

  2. Dependencies
    1. Documentation must be completed in standard order, as outlined in section 4.2

    1. All members of the CP317 class must have a position in the team, and the team may only include members of the CP317 class
    2. Budget is $0
    3. Project must be completed within the Spring 2016 WLU school term in order for marks to be submitted through LORIS.

4.1   Methods, Tools, and Techniques

The application will be written using the Ruby on Rails framework. The Ruby on Rails framework incorporates the Ruby language with HTML, CSS and Javascript. The front-end will integrate the Bootstrap framework. The unit tests will be written with RSpec, a Ruby testing framework.

4.2.3    Software Test Plan

The application will be tested through unit tests and by the SQA team. The unit tests will validate the operation of the models (entity objects - responsible for data persistence and retrieval). The SQA team will perform integration tests - validating the front-end interface works as expected.

4.3   User Documentation

The documentation will be developed on an on-going basis. Developers who write the code will be responsible for documenting the purpose of the code. Additionally, documentation/tutorials will be produced to demonstrate how to operate the website.
