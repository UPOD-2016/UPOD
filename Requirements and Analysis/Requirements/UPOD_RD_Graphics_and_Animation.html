1. Introduction

1.1 Scope

The UPOD Graphics group creates animations and interactive diagrams with SVG which allows users to gain a better understanding of Physics concepts. The specific diagrams that are animated are decided in tandem with the Physics Research group.

1.2 Definitions, Acronyms, and Abbreviations

MVC

  • A software architectural pattern for implementing user interfaces on computers. Traditionally used for GUIs.

    • Model (MVC)

  • Represents the equation or physics concept illustrated.

    • View (MVC)

  • Sliders, buttons, and other UI elements to interact with the model. Objects to reflect inputs and demonstrate the concepts.

    • Controller (MVC)

  • Liases and controls the view to reflect the model.

    • SVC

  • Scalable Vector Graphics.

  • IEEE Std 610.12-1990, IEEE Standard for Software Project Management Plans: http://bohr.wlu.ca/cp317/notes/IEEE_830.pdf
  • WLU CP317 Class of Spring 2013: Pong Tracker used as Example

    • 1.4 Overview

    The UPOD Graphics/Animation Requirement document outlines the following:

  • Overall description of the graphics
  • User's interaction with the graphics
  • The software used to create the graphics

    • 2. Overall Description

    2.1 Product Perspective

    Graphics uses the information gathered by the Physics Research group to create diagrams and animations. In addition, both groups collaborate to decide on what physics animations should be made and also what type of physics diagrams should be used.

    2.1.1 System Interfaces

    UPOD content operates on modern and standard browsers.

  • Google Chrome - version 51
  • Mozilla Firefox - version 47
  • Internet Explorer - version 11
  • Microsoft Edge - version 38
  • Safari - version 9.1

    • 2.1.2 User Interfaces

  • Users can use a mouse to interact with certain animations or diagrams on UPOD.
  • The keyboard can be used to input values to certain diagrams.

    • 2.1.3 Software Interfaces

  • Animations and graphics displaying the physics content are created using SVG and Javascript.

    • 2.1.4 Communications Interfaces

  • Graphics in UPOD is portable across the available internet browsers.

    • 2.1.5 Memory

  • Memory constraints should be expected while creating the graphics. More information shall be added later.

    • 2.2 Constraints

  • UPOD graphics must be added onto the site before the end of the 2016 spring term (August 10, 2016).
  • Budget is $0 so any software used to create the graphics must be free to use.

    • 3. Specific Requirements

    3.1 External Interfaces

  • Different javascript SVG animation libraries are being researched.

    • 3.2 Functions

  • Graphics to be included on the UPOD web interface.

    • 3.3 Performance Requirements

  • Graphics should be able to be seen on different browsers.
  • More detail in Analysis phase.

    • 3.4 Logical Database Requirements

  • All physics content and graphics will be stored on a relational database (see Back End Requirements).

    • 3.5 Software System Attributes

    3.5.1 Reliability

  • UPOD Graphics are expected to be used permanently in the site.

    • 3.5.2 Availability

  • Software used to create Graphics must be supported across the previously listed browsers.

    • 3.5.3 Security

  • All Graphics on UPOD are free to use for all and therefore, there are no major security constraints.

    • 3.5.4 Maintainability

  • Graphics should be coded to standard protocols and should be well documented for any future maintenance. UPOD is to be assumed available for the future; therefore having a set of individuals in charge of maintenance will not be constant.

    • 3.5.5 Portability

  • Graphics can be run on anything since it is using SVG. Any new software to be installed on hopper will be communicated to an Administrator once they are identified and validated.

    • 3.6 Organizing the Specific Requirements

    3.6.1 Functional Hierarchy

  • There is an object library that diagrams use. The library does not carry any pre-existing objects but rather objects(images) designed by the graphics group .
